# Тестирование совместимости PostgreSQL с SELECTOS

# Какие проверки реализованы

В скрипте test_postgres.sh:
Тест установки: Проверка корректного скачивания и установки пакета `postgresql` через `apt`.

Проверка работы службы: проверка того, что процесс поднят и имеет статус active.

Проверка сети: проверка, что субд успешно прослушивает порт `5432`.

Функциональный тест: Подключение к клиенту `psql` под пользователем `postgres` и создание тестовой базы данных и таблицы.

# Какие инструменты используются

Bash  для написания логики автотестов 

Systemd для проверки статуса демона СУБД.

ss  для проверки сетевых портов.

psql клиент PostgreSQL для отправки SQL-запросов.

# Как запустить проект локально

Я установила iso и поставила SelectOS на виртуальную машину

1. Склонируйте репозиторий на виртуальную машину:
   ```bash
   git clone 
   cd <название_папки>
Выдайте скрипту права на исполнение:
code
Bash
chmod +x pg_compat_test.sh
Запустите скрипт от имени суперпользователя (root), так как установка пакетов и проверка служб требуют повышенных привилегий:
code
Bash
sudo ./test_postgres.sh

В консоль будет выведен пошаговый отчет о выполнении тестов. При успешном прохождении напротив каждого пункта появится зеленая надпись PASS.
